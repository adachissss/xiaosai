# B 题第一小问建模计划

## 1. 题目要求理解

第一小问要求在不考虑预测误差的前提下，基于附件中 7 天、15 分钟分辨率的数据，为园区设计一套完整的源—荷—储—车协同运行方案。

方案需要给出每个时段的：

- 外网购电功率；
- 固定储能充放电功率；
- 电动车聚合充放电功率；
- 可调建筑负荷调整量；
- 弃光功率或售电功率。

同时，需要将所设计的协同运行方案与至少一种非协同运行方案进行比较，分析协同方案在运行成本、可再生能源消纳等方面的改进效果。

## 3. 总体建模思路

第一小问建议建立一个确定性线性规划模型。

其核心思想是：在已知未来 7 天光伏、负荷、电价、EV 到离站信息和电网约束的条件下，对固定储能、电动车、柔性建筑负荷和电网购售电进行统一优化调度。

优化目标是：

```text
最小化园区一周运行成本
```

同时兼顾：

- 提高光伏本地消纳；
- 减少高价时段购电；
- 满足 EV 离站电量需求；
- 满足建筑负荷运行需求；
- 满足电网购电与售电功率约束；
- 满足储能和 EV 电池能量约束。

## 4. 协同优化模型

### 4.1 集合与索引

设：

- `t = 1, 2, ..., T` 表示时间段，`T = 672`；
- `i = 1, 2, ..., N` 表示 EV 会话，`N = 102`；
- `b ∈ B` 表示建筑类型，包括办公楼、湿实验楼、教学中心。

时间步长：

```text
Δt = 0.25 h
```

### 4.2 决策变量

#### 4.2.1 电网变量

- `P_grid_buy[t]`：第 `t` 时段从电网购电功率；
- `P_grid_sell[t]`：第 `t` 时段向电网售电功率。

约束为：

```text
0 ≤ P_grid_buy[t] ≤ P_grid_import_limit[t]
0 ≤ P_grid_sell[t] ≤ P_grid_export_limit[t]
```

#### 4.2.2 光伏变量

- `P_pv_use[t]`：第 `t` 时段实际利用的光伏功率；
- `P_pv_curtail[t]`：第 `t` 时段弃光功率。

满足：

```text
P_pv_use[t] + P_pv_curtail[t] = P_pv_available[t]
P_pv_use[t] ≥ 0
P_pv_curtail[t] ≥ 0
```

#### 4.2.3 固定储能变量

- `P_bat_ch[t]`：固定储能充电功率；
- `P_bat_dis[t]`：固定储能放电功率；
- `E_bat[t]`：固定储能电量。

功率约束：

```text
0 ≤ P_bat_ch[t] ≤ 300
0 ≤ P_bat_dis[t] ≤ 300
```

电量约束：

```text
120 ≤ E_bat[t] ≤ 1080
```

状态转移方程：

```text
E_bat[t+1] = E_bat[t] + η_ch P_bat_ch[t] Δt - P_bat_dis[t] Δt / η_dis
```

其中：

```text
η_ch = η_dis = 0.95
```

初始条件：

```text
E_bat[1] = 600 kWh
```

#### 4.2.4 EV 变量

对每辆车 `i`、每个时段 `t` 定义：

- `P_ev_ch[i,t]`：EV 充电功率；
- `P_ev_dis[i,t]`：EV 反向放电功率；
- `E_ev[i,t]`：EV 电池电量。

若车辆不在站：

```text
P_ev_ch[i,t] = 0
P_ev_dis[i,t] = 0
```

若车辆在站：

```text
0 ≤ P_ev_ch[i,t] ≤ P_ev_ch_max[i]
0 ≤ P_ev_dis[i,t] ≤ P_ev_dis_max[i]
```

若该车辆不允许 V2B，则：

```text
P_ev_dis[i,t] = 0
```

EV 电池状态转移：

```text
E_ev[i,t+1] = E_ev[i,t] + η_ev_ch P_ev_ch[i,t] Δt - P_ev_dis[i,t] Δt / η_ev_dis
```

若题目未给出 EV 充放电效率，可假设：

```text
η_ev_ch = η_ev_dis = 0.95
```

离站需求约束：

```text
E_ev[i, departure_i] ≥ E_required[i]
```

电池容量约束：

```text
0 ≤ E_ev[i,t] ≤ E_ev_capacity[i]
```

聚合 EV 功率为：

```text
P_EV_ch_total[t] = Σ_i P_ev_ch[i,t]
P_EV_dis_total[t] = Σ_i P_ev_dis[i,t]
P_EV_net[t] = P_EV_ch_total[t] - P_EV_dis_total[t]
```

其中 `P_EV_net[t] > 0` 表示 EV 聚合充电，`P_EV_net[t] < 0` 表示 EV 聚合向园区供电。

#### 4.2.5 建筑柔性负荷变量

对每类建筑 `b`、每个时段 `t` 定义：

- `P_shift_down[b,t]`：从该时段转移出去的负荷；
- `P_shift_up[b,t]`：转移到该时段补回的负荷；
- `P_shed[b,t]`：该时段削减的负荷。

约束为：

```text
0 ≤ P_shift_down[b,t] ≤ P_shiftable_max[b]
0 ≤ P_shift_up[b,t] ≤ rebound_factor[b] × P_shiftable_max[b]
0 ≤ P_shed[b,t] ≤ P_sheddable_max[b]
```

为了避免跨天转移过远，可设置每日能量守恒：

```text
Σ_{t ∈ d} P_shift_up[b,t] Δt = rebound_factor[b] × Σ_{t ∈ d} P_shift_down[b,t] Δt
```

调整后的建筑总负荷为：

```text
P_load_adj[t]
= P_native_load[t]
- Σ_b P_shift_down[b,t]
- Σ_b P_shed[b,t]
+ Σ_b P_shift_up[b,t]
```

---

## 5. 功率平衡约束

每个时段都需要满足园区功率平衡：

```text
P_pv_use[t]
+ P_grid_buy[t]
+ P_bat_dis[t]
+ Σ_i P_ev_dis[i,t]
=
P_load_adj[t]
+ P_bat_ch[t]
+ Σ_i P_ev_ch[i,t]
+ P_grid_sell[t]
```

即：

```text
光伏利用 + 电网购电 + 储能放电 + EV 放电
=
调整后建筑负荷 + 储能充电 + EV 充电 + 电网售电
```

---

## 6. 目标函数

第一小问不考虑固定储能和 EV 电池寿命损耗，因此目标函数主要包括：

1. 电网购电费用；
2. 电网售电收益；
3. 建筑负荷削减惩罚；
4. 少量弃光惩罚。

目标函数为：

```text
min Σ_t [
  c_buy[t] P_grid_buy[t] Δt
- c_sell[t] P_grid_sell[t] Δt
+ Σ_b c_shed[b] P_shed[b,t] Δt
+ c_shift Σ_b (P_shift_down[b,t] + P_shift_up[b,t]) Δt
+ c_curtail P_pv_curtail[t] Δt
]
```

其中：

- `c_buy[t]` 为电网购电价；
- `c_sell[t]` 为电网售电价；
- `c_shed[b]` 为不同建筑负荷削减惩罚；
- `c_shift` 为可转移负荷舒适度惩罚，可取较小值，例如 `0.05 元/kWh`；
- `c_curtail` 为弃光惩罚，可取较小值，例如 `0.01 元/kWh`。

设置弃光惩罚的目的不是显著增加成本，而是在经济性相近的方案中优先选择光伏消纳更高的方案。

## 7. 非协同基准方案设计

为了体现协同优化的改进效果，需要设计一个非协同运行方案作为 baseline。

推荐的非协同方案如下：

### 7.1 建筑负荷

建筑负荷不参与调节：

```text
P_load_adj[t] = P_native_load[t]
```

即：

```text
P_shift_down[b,t] = 0
P_shift_up[b,t] = 0
P_shed[b,t] = 0
```

### 7.2 电动车

EV 到站后立即按照最大充电功率充电，直到达到离站需求电量。

EV 不参与反向供电：

```text
P_ev_dis[i,t] = 0
```

### 7.3 固定储能

固定储能采用简单规则运行，而不进行全局优化。

建议规则为：

1. 当光伏出力大于建筑负荷和 EV 即时充电负荷时，用富余光伏给固定储能充电；
2. 当电价处于高价时段且储能电量充足时，固定储能放电补充园区负荷；
3. 其他时段储能不动作；
4. 不为了售电而主动放电。

高电价阈值可以取：

```text
c_buy[t] ≥ 0.9 元/kWh
```

该 baseline 具有一定现实合理性，但不具备跨时段、跨资源的协同优化能力。

## 8. 对比评价指标

协同方案与非协同方案从以下指标进行比较。

### 8.1 总运行成本

```text
Cost = 购电费用 - 售电收益 + 负荷削减惩罚 + 负荷转移惩罚 + 弃光惩罚
```

成本降低率：

```text
成本降低率 = (Cost_baseline - Cost_coordinated) / Cost_baseline × 100%
```

### 8.2 光伏消纳率

```text
PV_consumption_rate = Σ_t P_pv_use[t] Δt / Σ_t P_pv_available[t] Δt
```

弃光率：

```text
PV_curtailment_rate = Σ_t P_pv_curtail[t] Δt / Σ_t P_pv_available[t] Δt
```

### 8.3 电网购电量

```text
Grid_import_energy = Σ_t P_grid_buy[t] Δt
```

### 8.4 电网售电量

```text
Grid_export_energy = Σ_t P_grid_sell[t] Δt
```

### 8.5 峰值购电功率

```text
Peak_import = max_t P_grid_buy[t]
```

重点分析特殊约束时段：

- 2025-07-17 13:00-16:00，购电上限降低至 650 kW；
- 2025-07-18 17:00-19:00，购电上限降低至 700 kW。

### 8.6 EV 服务质量

检查所有车辆是否满足：

```text
E_ev[i, departure_i] ≥ E_required[i]
```

统计：

- EV 离站需求满足率；
- EV 总充电量；
- EV 总放电量；
- 参与 V2B 的车辆数量。

### 8.7 建筑负荷调节量

统计：

- 总削减电量；
- 总转移电量；
- 各建筑类型的调节贡献；
- 调节主要发生在哪些时段。

## 11. 论文中第一问建议写法

### 11.1 问题分析

第一问可以表述为一个确定性多资源协同调度问题。园区同时具有光伏、固定储能、电动车、柔性建筑负荷和外部电网。各资源之间存在明显的时序耦合关系：光伏出力集中在白天，电价具有峰谷差异，EV 在白天集中到站并在傍晚离站，建筑负荷具有可削减和可转移能力。因此，需要建立统一优化模型，而不是分别对各设备进行独立控制。

### 11.2 模型假设

建议采用以下假设：

1. 不考虑预测误差，所有光伏、负荷、电价、EV 到离站数据均已知；
2. 15 分钟内各功率变量保持恒定；
3. 固定储能和 EV 充放电效率为常数；
4. EV 到站和离站时间严格按照数据执行；
5. 第一问不考虑电池寿命损耗；
6. 可转移建筑负荷在同一天内完成补偿；
7. 光伏可用于本地负荷、储能充电、EV 充电、售电，也可在必要时弃光；
8. 不考虑设备启停成本和通信延迟。

### 11.3 模型建立

论文中可按以下顺序建立模型：

1. 光伏利用与弃光模型；
2. 固定储能充放电模型；
3. EV 车辆级充放电模型；
4. 建筑柔性负荷模型；
5. 园区功率平衡模型；
6. 电网购售电约束；
7. 运行成本最小化目标函数。

### 11.4 结果分析

结果分析重点包括：

1. 协同方案如何利用低价时段和光伏高发时段进行充电；
2. 协同方案如何在高价时段或购电受限时段释放储能和 EV 灵活性；
3. 建筑柔性负荷如何从高价或受限时段转移到低价或光伏富余时段；
4. 协同方案相对于非协同方案在成本、峰值购电、光伏消纳方面的改善；
5. 所有 EV 是否均满足离站电量需求。

---

## 12. 第一问最终方案定位

第一小问建议最终采用：

```text
确定性线性规划协同优化模型 + 规则型非协同 baseline 对比
```

该方案的优点是：

1. 数学模型完整，适合写入数学建模论文；
2. 结果可通过程序稳定复现；
3. 可以自然输出每个时段的完整运行方案；
4. 对比方案清晰，能够体现协同运行价值；
5. 后续第二问可以直接加入固定储能和 EV 电池寿命成本；
6. 后续第三问可以继续加入碳排放约束和碳交易成本。

因此，第一问应作为整篇论文的基础模型，为第二问和第三问提供统一的建模框架。
