# 第三问分析思路：考虑碳排放约束的园区协同调度

## 1. 问题重述

> 在问题二的基础上，假设园区需满足一个为期一周的总碳排放上限，且可在允许范围内通过碳交易市场购买或出售排放额度。
>
> 1. 给定不同的碳排放上限约束，系统运行方案及设备调度安排会发生怎样的变化；
> 2. 碳交易价格水平如何影响园区从外部购电、储能充放电以及电动车参与反向供电的决策；
> 3. 如何兼顾经济性、电池寿命与碳排放约束，形成一套综合运行策略。

**核心认知**：碳交易是 cap-and-trade，碳上限（cap）和碳价（price）必须**同时存在**。不能拆成"只有 cap 的命令控制"或"只有 price 的碳税"。

## 2. 数据前提

| 参数 | 数据中是否提供 | 说明 |
|------|:---:|------|
| 碳强度 `grid_carbon_kg_per_kwh` | **给了** | `timeseries_15min.csv`，逐时段 0.55–0.74 kg/kWh，夜间高白天低，波动幅度 ~35% |
| 碳排放上限 | **没给** | 需自行假设 |
| 碳交易价格 | **没给** | 需自行假设，参考中国碳市场（~100 元/吨）和 EU ETS（~500 元/吨） |

## 3. 碳交易机制建模

### 碳排放量

```
碳排放[t] = grid_buy[t] × grid_carbon_kg_per_kwh[t] × Δt
全周总碳排放 = Σ_t 碳排放[t]
```

售电不计入碳减排（Scope 2 标准）。

### 碳交易成本

```
碳交易净成本 = carbon_price × (实际排放 − 免费配额)
```

其中免费配额 = 碳排放上限（cap）。排放 < cap 时成本为负（出售配额获利）。

### 场景设计

**碳价场景**：

| 场景 | 碳价（元/吨） | 参考 |
|------|------:|------|
| 无碳成本 | 0 | 基准 |
| 低碳价 | 100 | 中国碳市场 |
| 中碳价 | 300 | 中间情景 |
| 高碳价 | 500 | EU ETS |

**碳配额场景**（数据未给定，以自然排放 E_nat 为锚点）：

| 场景 | cap 占比 | 含义 |
|------|:---:|------|
| 无约束 | ∞ | 基准 |
| 宽松 | 105% | 不绑定 |
| 基准 | 100% | 等于自然排放 |
| 严格 | 93% | 需减排 7% |
| 极严 | 86% | 需减排 14% |

注：宽松/基准 cap ≥ 自然排放，数学上必然不绑定，不是分析发现。

## 4. S5 优化模型

在 S4（运行成本 + 固定储能寿命 + EV 寿命）基础上：

**目标函数**：

```
min 运行成本 + 固定储能寿命损耗 + EV 寿命损耗 + 碳交易成本
```

碳交易成本在 LP 中实现为购电系数的加成：

```
c[grid_buy[t]] += carbon_price × carbon_intensity[t] × Δt
```

**可选约束**：全周碳排放上限 `Σ(grid_buy × CI × Δt) ≤ cap`。

**碳约束影子价格**：LP 求解后从对偶变量提取，代表边际减排成本（元/kg）。它是连接 Q1（cap）和 Q2（price）的桥梁——碳价需超过影子价格才能驱动主动减排。
