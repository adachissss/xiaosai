# 第三问第 1 小问解决思路：不同碳排放上限对系统调度的影响

## 1. 小问目标

第三问第 1 小问要求：

```text
分析不同的碳排放上限（cap）下，系统运行方案及设备调度安排会发生怎样的变化。
```

因此本小问的核心任务是：以 S5 模型为基础，保持碳价为零（仅通过硬约束），逐步收紧碳排放上限，观察系统调度行为的适应性调整。

---

## 2. 模型基础

本小问使用的 S5 模型建立在第二问 S4 模型之上。

S4 目标函数：

```text
min 运行成本 + 固定储能寿命损耗 + EV 寿命损耗
```

S5 在 S4 基础上增加碳交易成本到目标函数，以及可选的全周碳排放上限约束：

```text
min 运行成本 + 固定储能寿命损耗 + EV 寿命损耗 + 碳交易成本

s.t. Σ_t(grid_buy[t] × carbon_intensity[t] × Δt) ≤ cap
```

本小问中设置 `carbon_price = 0`，仅通过 `cap` 参数约束碳排放。

---

## 3. 碳排放计算方法

### 3.1 碳排放来源

园区碳排放仅来自外网购电。光伏、固定储能和 EV 运行本身不产生直接碳排放。

### 3.2 逐时段碳核算

```text
carbon_emission[t] = grid_buy[t] × grid_carbon_kg_per_kwh[t] × Δt
```

其中 `grid_carbon_kg_per_kwh[t]` 来自 `B/B_data/timeseries_15min.csv`，范围为 0.55 ~ 0.74 kg CO2/kWh，周均 0.651 kg/kWh。高碳强度时段（火电占比高）通常出现在早晚高峰，低碳强度时段（光伏出力多）出现在白天。

### 3.3 全周碳排放

```text
Total_Carbon = Σ_t carbon_emission[t]
```

## 4. 场景设计

先以 `carbon_price = 0`、`cap = None` 运行 S5，得到自然排放基准值：

```text
baseline_carbon = 42237.3 kg CO2/周
```

以此为 100%，设计五级 cap 场景：

| 场景 | cap (kg CO2/周) | 占基准值比例 | 含义 |
|---:|---:|---:|---|
| 无约束 | ∞ | – | 无碳排放上限，仅靠价格信号 |
| 宽松 | 44349 | 105% | 比自然排放高 5%，实际上不绑定 |
| 基准 | 42237 | 100% | 等于自然排放，恰好不绑定 |
| 严格 | 39281 | 93% | 要求减排 7%，开始绑定 |
| 极严 | 36324 | 86% | 要求减排 14%，强约束 |

设计逻辑：
- 105% 和 100% cap 作为对照，验证系统自然排放水平是否低于这些 cap
- 93% 和 86% 作为递进收紧，观察从"恰好绑定"到"强约束"的过渡

## 5. 关键代码说明

### 5.1 碳排放上限约束实现

在 `carbon_aware_strategy.py:219-223`：

```python
if carbon_cap_kg is not None and carbon_cap_kg < float("inf"):
    carbon_row: dict[int, float] = {}
    for t in range(T):
        carbon_row[grid_buy[t]] = carbon_intensity[t] * DT
    ub(carbon_row, carbon_cap_kg)
```

含义：将全周购电产生的碳排放用一条线性不等式约束加入 LP。每个时段 `grid_buy[t]` 的系数是 `carbon_intensity[t] × DT`（该时段的单位购电碳排放量）。

含义：对每个 cap 值调用 S5 求解器，提取运行成本、碳排放、储能吞吐量等指标进行横向对比。碳价固定为零，确保差异仅来自 cap 约束。

### 5.3 碳排放计算

在 `solve_problem3.py:65-68`：

```python
def compute_carbon_footprint(schedule, data_dir):
    ts = pd.read_csv(data_dir / "timeseries_15min.csv")
    ci = ts["grid_carbon_kg_per_kwh"].to_numpy()
    return float(np.dot(schedule["grid_buy_kw"].to_numpy(), ci * DT))
```

含义：对每个调度方案的购电功率向量，逐时段乘以碳强度再累加，得到全周实际碳排放。

## 7. 结果分析

### 7.1 无约束/宽松/基准三个场景结果一致

宽松 cap（105%）和基准 cap（100%）均未实际约束系统——系统在没有碳上限时的自然排放（42237 kg）已经低于或等于这些 cap，因此 LP 的最优解不变。

这说明在当前园区配置（大量光伏 + 储能 + EV 调节）下，碳排放水平相对较低，轻度碳约束不起实际作用。

### 7.2 严格 cap（93%）：运行成本上升 4.4%

当 cap 收紧到 39281 kg（93% 基准）时，LP 被迫减少外网购电约 6%。运行成本从 44485 元上升到 46425 元（+1940 元），但做到了减排 7%。

同时出现以下变化：
- 负荷削减从 693 kWh 增加到 4357 kWh——系统主动削减了柔性负荷
- 负荷转移下调从 5649 kWh 减少到 3988 kWh
- 这说明 cap 收紧使得建筑侧的节能响应增强

### 7.3 极严 cap（86%）：运行成本上升 13.4%

当 cap 收紧到 36324 kg（86%）时：
- 购电总量从 63990 kWh 降至 56216 kWh（-12%）
- 运行成本从 44485 元大幅上升到 50436 元（+5951 元）
- 峰值购电从 608 kW 降至 496 kW——通过减少峰值购电来降低碳排放
- 负荷削减大幅增加到 8366 kWh——柔性负荷成为减排的重要调节手段
- 负荷转移下调减少到 4175 kWh

### 7.4 电池调度行为的变化

| 指标 | 无约束 | 严格 | 极严 | 变化 |
|---:|---:|---:|---:|
| 固定储能吞吐 kWh | 12952 | 12879 | 12773 | ↓ 179 |
| EV 充电 kWh | 2581 | 2581 | 2579 | ≈ -2 |
| EV 放电 kWh | 434 | 434 | 432 | ≈ -2 |
| PV 消纳率 | 100% | 100% | 100% | 不变 |

关键发现：
1. **固定储能吞吐量随 cap 收紧略有下降**（-1.4%），而不是上升。表面看这有些反直觉——减排应该"调动更多本地资源"。实际原因是：储能本身的充电需要从电网取电，而在碳排放受严格约束时，从高碳强度时段充电会产生较多碳排放，LP 因此减少了储能的使用频率。
2. **EV 调度基本不变**——EV 吞吐量从 3015 kWh 仅降至 3011 kWh，变化幅度极小。EV 的 V2B 放电量为 434 kWh，在严格和极严 cap 下基本不变。原因是 EV 的寿命损耗成本仍是主要制约因素。
3. **光伏消纳率始终保持 100%**——碳排放约束没有以牺牲光伏利用为代价。

### 7.5 综合成本拆分

| 成本项 | 无约束 | 严格 | 极严 |
|---:|---:|---:|
| 运行成本 元 | 44485 | 46425 | 50436 |
| 固定储能寿命损耗 元 | 712 | 708 | 703 |
| EV 寿命损耗 元 | 261 | 261 | 260 |
| 碳交易成本 元 | 0 | 0 | 0 |
| 综合成本 元 | 45458 | 47394 | 51399 |

碳排放的收紧主要增加了运行成本（来自购电替代和负荷削减惩罚），电池寿命损耗成本反而略有下降。

## 9. 本小问结论

第三问第 1 小问已完成：

```text
以 S5 碳感知模型为基础，在 carbon_price = 0 的条件下，设定了无约束/宽松/基准/严格/极严
五级碳排放上限，分析了不同 cap 对系统调度行为的影响。
```

核心发现：
1. 105% 和 100% 基准的 cap 不产生约束力——园区自然排放低于这些水平
2. 93% cap 开始绑定：运行成本上升 4.4%，减排 7%
3. 86% cap 较强约束：运行成本上升 13.4%，减排 14%
4. 减排主要手段是减少外网购电 + 增加负荷削减，EV 调度变化很小
5. 光伏消纳率始终保持 100%，碳约束没有以牺牲光伏为代价
6. 固定储能吞吐量反而略有下降，因为从高碳强度电网充电会产生间接碳排放
