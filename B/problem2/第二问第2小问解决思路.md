# 第二问第 2 小问解决思路：考虑电池寿命损耗的协同调度

## 1. 小问目标

第二问第 2 小问要求：

```text
在运行方案中同时考虑运行成本与电池寿命损耗，重新设计调度安排。
```

因此本小问不是单纯计算寿命损耗，而是要把第 1 小问建立的寿命损耗成本正式加入优化目标函数，重新求解调度方案。

本文将第一问的 `S3_full_coordination` 作为未考虑寿命损耗的参考方案，建立新的：

```text
S4_degradation_aware：考虑寿命损耗的完整协同优化方案
```

## 2. 模型基础

S4 基于第一问 S3 完整协同模型。

S3 已经同时考虑：

1. 外网购电和售电；
2. 光伏利用和弃光；
3. 固定储能充放电；
4. EV 智能充电和 V2B 放电；
5. 建筑柔性负荷转移和削减；
6. 峰值购电功率控制；
7. EV 离站电量需求约束。

第 2 小问保持这些约束不变，只改变目标函数。这样可以保证 S3 与 S4 的差异主要来自“是否计入电池寿命损耗”。

---

## 3. 新目标函数

S3 的目标函数可以概括为：

```text
min 运行成本
```

S4 的目标函数改为：

```text
min 运行成本 + 固定储能寿命损耗成本 + EV 寿命损耗成本
```

即：

```text
min C_operation + C_bat_deg + C_ev_deg
```

其中运行成本包括购电成本、售电收益、弃光惩罚、负荷转移舒适度惩罚、负荷削减惩罚、未满足负荷惩罚、EV 离站电量缺口惩罚和峰值购电惩罚。

---

## 4. 固定储能寿命损耗加入方式

固定储能等效吞吐量为：

```text
E_bat_throughput = Σ_t(P_bat_ch[t] + P_bat_dis[t]) × Δt
```

固定储能寿命损耗成本为：

```text
C_bat_deg = c_bat_deg × E_bat_throughput
```

附件给定：

```text
c_bat_deg = 0.055 元/kWh
```

因此在 LP 目标函数中，对每个时段的固定储能充放电变量加入线性成本：

```text
c[bat_ch[t]]  += c_bat_deg × Δt
c[bat_dis[t]] += c_bat_deg × Δt
```

由于 `P_bat_ch[t]` 和 `P_bat_dis[t]` 是功率变量，乘以 `Δt = 0.25 h` 后才是电量，因此目标函数系数必须乘以 `Δt`。

---

## 5. EV 寿命损耗加入方式

第 1 小问由于只读取第一问聚合结果，只能用 EV 加权平均单位寿命成本近似估算。

第 2 小问重新建立 LP，因此保留了每辆 EV 的逐时段充放电变量：

```text
P_ev_ch[i,t]
P_ev_dis[i,t]
```

所以 EV 寿命损耗可以精确写为：

```text
C_ev_deg = Σ_i c_ev_deg[i] × Σ_t(P_ev_ch[i,t] + P_ev_dis[i,t]) × Δt
```

其中 `c_ev_deg[i]` 来自 `B/B_data/ev_sessions.csv` 的字段：

```text
degradation_cost_cny_per_kwh_throughput
```

在 LP 目标函数中，对每辆车分别加入：

```text
c[ev_ch[i,t]]  += c_ev_deg[i] × Δt
c[ev_dis[i,t]] += c_ev_deg[i] × Δt
```

这样，寿命成本高的 EV 会更少参与无必要的 V2B；只有当放电带来的运行收益高于其寿命损耗时，模型才会调用 EV 放电。

## 7. 关键代码说明

### 7.1 固定储能寿命成本加入目标函数

```python
battery_deg_cost = float(
    data.asset["stationary_battery_degradation_cost_cny_per_kwh_throughput"]
)
c[bat_ch] += battery_deg_cost * DT
c[bat_dis] += battery_deg_cost * DT
```

含义：从附件读取固定储能单位吞吐寿命成本，并把它作为充放电功率变量的线性成本系数加入 LP 目标函数。

### 7.2 EV 寿命成本加入目标函数

```python
ev_cost_col = "degradation_cost_cny_per_kwh_throughput"
ev_unit_cost = ev[ev_cost_col].astype(float).to_numpy()
for n, unit_cost in enumerate(ev_unit_cost):
    c[ev_ch[n, :]] += unit_cost * DT
    c[ev_dis[n, :]] += unit_cost * DT
```

含义：每辆 EV 使用自己的单位寿命成本，因此第 2 小问不再使用普通平均或加权平均近似，而是车辆级精确计入目标函数。

### 7.3 S4 EV 结果输出

```python
ev_result["charge_throughput_kwh"] = x[ev_ch].sum(axis=1) * DT
ev_result["discharge_throughput_kwh"] = x[ev_dis].sum(axis=1) * DT
ev_result["total_throughput_kwh"] = (
    ev_result["charge_throughput_kwh"]
    + ev_result["discharge_throughput_kwh"]
)
ev_result["unit_degradation_cost_cny_per_kwh"] = ev_unit_cost
ev_result["degradation_cost_cny"] = (
    ev_result["total_throughput_kwh"]
    * ev_result["unit_degradation_cost_cny_per_kwh"]
)
```

作用：记录每辆 EV 的充电吞吐量、放电吞吐量、总吞吐量、单位寿命成本和寿命损耗成本。

## 9. 当前数值结果

基于当前数据，S3 与 S4 的核心结果如下：

| 指标 | S3 未考虑寿命 | S4 考虑寿命 | S4-S3 |
|---|---:|---:|---:|
| 运行成本/元 | 44474.640 | 44485.204 | 10.564 |
| 固定储能寿命损耗成本/元 | 712.383 | 712.383 | 0.000 |
| EV 寿命损耗成本/元 | 288.246 | 260.826 | -27.421 |
| 总寿命损耗成本/元 | 1000.629 | 973.209 | -27.421 |
| 综合成本/元 | 45475.270 | 45458.413 | -16.857 |
| 固定储能吞吐量/kWh | 12952.421 | 12952.421 | 0.000 |
| EV 吞吐量/kWh | 3349.736 | 3015.055 | -334.682 |

| 峰值购电功率/kW | 607.550 | 607.550 | 0.000 |

可以看出：

1. S4 的运行成本比 S3 上升约 `10.564 元`；
2. S4 的 EV 寿命损耗成本下降约 `27.421 元`；
3. S4 的综合成本下降约 `16.857 元`；
4. S4 明显减少了 EV 充放电吞吐量，尤其减少了 V2B 放电量；
5. 固定储能吞吐量在当前参数下没有变化，说明固定储能调节收益仍足以覆盖其寿命成本。

## 11. 本小问结论

第二问第 2 小问已经完成：

```text
在 S3 完整协同优化模型基础上加入电池寿命损耗成本，建立 S4_degradation_aware 调度方案。
```

结果表明，引入寿命损耗后，模型会减少 EV 的不必要充放电和 V2B 调用，使 EV 吞吐量和 EV 寿命损耗成本下降。虽然短期运行成本略有上升，但总寿命损耗成本下降更多，最终综合成本低于 S3。

因此，S4 更能体现短期运行经济性与长期电池健康之间的权衡。
