# 第三问第 1 小问解决思路：不同碳排放上限对系统调度的影响

## 1. 小问目标

第三问第 1 小问要求：

```text
分析不同的碳排放上限（cap）下，系统运行方案及设备调度安排会发生怎样的变化。
```

因此本小问的核心任务是：以 S5 模型为基础，保持碳价为零（仅通过硬约束），逐步收紧碳排放上限，观察系统调度行为的适应性调整。

---

## 2. 模型基础：cap-and-trade（碳配额交易）

本小问使用的 S5 模型建立在第二问 S4 模型之上。

S4 目标函数：

```text
min 运行成本 + 固定储能寿命损耗 + EV 寿命损耗
```

S5 采用 cap-and-trade 机制：

- 园区获得免费碳排放配额（free_allowance）
- 实际排放 ＞ 配额 → 购买差额，碳交易成本为正
- 实际排放 ＜ 配额 → 卖出剩余配额获利，碳交易成本为负（收入）
- 碳交易成本 = carbon_price × (total_emissions - free_allowance)，可为负

在 LP 目标函数中：

```text
min 运行成本 + 固定储能寿命损耗 + EV 寿命损耗 + carbon_price × total_emissions
```

free_allowance 是常量，不影响优化（仅影响事后成本核算）。**碳价才是唯一影响调度行为的参数。**

本小问中固定 `carbon_price = 100 元/吨`，变化 free_allowance。

---

## 3. 碳排放计算方法

### 3.1 碳排放来源

园区碳排放仅来自外网购电。光伏、固定储能和 EV 运行本身不产生直接碳排放。

### 3.2 逐时段碳核算

```text
carbon_emission[t] = grid_buy[t] × grid_carbon_kg_per_kwh[t] × Δt
```

其中 `grid_carbon_kg_per_kwh[t]` 来自 `B/B_data/timeseries_15min.csv`，范围为 0.55 ~ 0.74 kg CO2/kWh，周均 0.651 kg/kWh。高碳强度时段（火电占比高）通常出现在早晚高峰，低碳强度时段（光伏出力多）出现在白天。

### 3.3 全周碳排放

```text
Total_Carbon = Σ_t carbon_emission[t]
```

## 4. 场景设计

先以 `carbon_price = 0`、`free_allowance = None` 运行 S5，得到自然排放基准值：

```text
baseline_carbon = 42237.3 kg CO2/周
```

以此为 100%，设计五级配额场景（固定碳价=100元/吨）：

| 场景 | free_allowance (kg CO2/周) | 占基准值比例 | 含义 |
|---:|---:|---:|---|
| 无配额 | ∞（无免费配额） | – | 所有排放均需购买配额 |
| 宽松配额 | 44349 | 105% | 配额高于实际排放，可卖出获利 |
| 基准配额 | 42237 | 100% | 配额≈实际排放，接近零交易 |
| 紧缩配额 | 39281 | 93% | 配额低于实际排放，需购买差额 |
| 极紧缩配额 | 36324 | 86% | 配额远低于实际排放，大量购买 |

设计逻辑：
- 在 cap-and-trade 下，配额是常量，**所有场景调度结果完全相同**（碳排放均为 42126 kg）
- 场景之间的唯一差异是碳交易成本（收入 vs 支出）
- 通过对比展示"配额分配不影响调度，只影响财务"这一核心结论

## 5. 关键代码说明

### 5.1 碳交易成本加入目标函数

在 `carbon_aware_strategy.py:117-119`：

```python
carbon_intensity = ts["grid_carbon_kg_per_kwh"].to_numpy()
c[grid_buy] += carbon_price_cny_per_kg * carbon_intensity * DT
```

含义：每个时段购电的边际成本 = 购电价 + 碳价 × 碳强度。碳价越高，高碳强度时段购电越贵。

### 5.2 无硬上限约束

在 cap-and-trade 下，没有"硬碳排放上限"——排放可以通过购买配额来覆盖。因此删除了旧版中的 `ub(carbon_row, cap)` 约束。调度行为的唯一碳驱动因素是价格信号。

### 5.3 碳交易成本计算修正

在 `carbon_aware_strategy.py` 中，事后成本核算改为：

```python
if free_allowance_kg is not None:
    carbon_cost_cny = carbon_price_cny_per_kg * (total_carbon_kg - free_allowance_kg)
else:
    carbon_cost_cny = carbon_price_cny_per_kg * total_carbon_kg
```

含义：碳交易成本 = 碳价 × (实际排放 - 免费配额)。当实际排放低于配额时，成本为负（卖配额收入）。

### 5.4 场景遍历

在 `solve_problem3.py` 中：

```python
for label, allowance in allowance_scenarios:
    sched, ev_r, info = build_and_solve_carbon_aware(
        data, carbon_price_cny_per_kg=0.10, free_allowance_kg=allowance,
        scheme_name=f"S5_cap_{label}"
    )
```

含义：固定碳价 100 元/吨，遍历不同配额水平。由于配额不参与优化，所有场景得到的调度方案完全相同。

## 7. 结果分析

### 7.1 核心发现：配额不影响调度

在 cap-and-trade 机制下，免费配额是常数项，在目标函数中不产生梯度。因此**不同配额水平下的系统调度方案完全相同**。这是一个重要的理论结果：碳配额分配是一种**财富转移工具**，而非**行为调节工具**。

### 7.2 碳价是唯一的行为驱动因素

调度方案的唯一碳相关驱动因素是碳价（carbon_price）。碳价 100 元/吨时，碳排放从无碳约束时的 42237 kg 降至 42126 kg（-111 kg），说明碳价信号已经开始发挥作用。

### 7.3 配额对财务的影响

| 场景 | 碳交易成本 | 园区的市场角色 |
|---:|---|
| 无配额 | +4213 元（支出） | 净买家——全部排放均需购买配额 |
| 宽松配额（105%） | -222 元（收入） | 净卖家——配额充足，可卖出获利 |
| 基准配额（100%） | -11 元（收入） | 接近平衡 |
| 紧缩配额（93%） | +285 元（支出） | 净买家——配额不足 |
| 极紧缩配额（86%） | +580 元（支出） | 净买家——配额严重不足 |

综合成本的差异完全来自碳交易成本的差异。宽松配额时综合成本 45236 元（最低），无配额时 49671 元（最高），差值 4435 元。

### 7.4 政策含义

1. **碳价设定**决定了园区的减排力度——碳价越高，减排越多
2. **配额分配**决定了园区的财务位置——配额越宽松，碳交易收入越高（或支出越低）
3. 两者应**配合使用**：合理的碳价驱动减排行为，适度的配额保障财务稳健

## 9. 本小问结论

第三问第 1 小问已完成：

```text
在 cap-and-trade 机制下，固定碳价 100 元/吨，设定了五级免费碳排放配额
（无配额/宽松/基准/紧缩/极紧缩），分析了不同配额对系统的影响。
```

核心发现：
1. **碳配额不影响调度行为**——所有五级配额下，系统调度方案完全相同（排放 42126 kg），因为配额是常数项，不产生优化梯度
2. **碳价是唯一的行为驱动因素**——碳价 100 元/吨使排放从无碳约束的 42237 kg 降至 42126 kg
3. **碳配额决定园区的财务位置**——宽松配额时可卖配额赚 222 元，无配额时需花 4213 元购买
4. **碳配额是一种财富分配工具，碳价是一种行为调节工具**——两者应配合使用
5. 综合成本从 45236 元（宽松配额）到 49671 元（无配额），差额全来自碳交易
